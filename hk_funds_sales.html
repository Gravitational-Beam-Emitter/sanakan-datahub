KYP Compliance — The Bottleneck

SFC Code of Conduct §5.1A / §5.5 mandate rigorous product due diligence. Traditional manual KYP workflows fail to scale.

SYS.ERR: Data Fragmentation

Fund documents scattered across 50+ manager websites, SFC registers, exchange bulletins. Manual collection takes 2–4 hours per fund. Covering a full product shelf requires months of analyst time.

SYS.ERR: Rating Inconsistency

Different RMs produce different conclusions for the same manager. No standardized scoring framework. Compliance audit trails are non-existent and difficult to defend during SFC inspections.

SYS.ERR: Monitoring Blind Spots

KYP is not one-shot. NAV changes, manager violations, fund document updates require continuous monitoring. Manual tracking misses critical events, triggering regulatory exposure.

SYS.ERR: Complex Product Classification

§5.5 lists 14 complex product types. §5.1A identifies 71 derivative funds. Manual classification of each fund is error-prone and operationally inefficient.

Frequently Asked Questions

Where does the data come from? How often is it updated?

Data sources: SFC Public Register (daily sync), 50+ manager websites (daily scrape), Yahoo Finance (daily NAV), SFC Enforcement Notices (real-time), HKEX (periodic), Webb-Site and Wikipedia (weekly). Core data refreshed daily, fully automated.

Can the rating model be customized?

Fully customizable. Enterprise users can create custom rating templates: adjust 18 factor weights, modify scoring thresholds (F1–F5 boundaries), select different scoring functions. Six preset templates including Broker 18I provided as starting points. All changes take effect immediately with historical ratings preserved for audit.

How is data accuracy ensured?

(1) Multi-source cross-validation: ISIN triple-checked across SFC register, manager site, and Yahoo Finance. (2) All auto-parsed results retain original document links for manual verification. (3) Every rating score has explicit source fields and computation logic, fully auditable. (4) Data update timestamps transparently displayed.

Can this be deployed on-premise?

Yes. Enterprise supports on-premise private deployment: all data collection, storage, and API services run inside your corporate network. Data never leaves your infrastructure. Private cloud deployment on AWS, Azure, or Alibaba Cloud also supported.

What fund types are covered?

All SFC-authorized public funds (2,021): equity, bond, balanced, money market, ETFs, structured products, L&I products, synthetic ETFs, and 154 public OFC sub-funds. Additionally: 2,231 private OFCs and non-authorized funds for PI-only distribution.
