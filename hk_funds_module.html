9. Deployment

Server Architecture

Server: /opt/eco-data/ ├── hk_funds/ # Python backend module │ ├── api.py # FastAPI server (port 8004) │ ├── rating_engine.py # Template-driven scoring engine (1,592 lines) │ ├── storage.py # DuckDB storage layer (2,507 lines) │ ├── manager_scoring.py # Internal Control scoring │ ├── manager_connectors/ # 51 manager website scrapers │ └── hk_funds.duckdb # Database (842 MB, 22 tables) ├── mcp/eco_data_server.py # MCP server (2,638 lines, 27 HK tools) ├── eco_data_sdk/ # Python client SDK └── frontend/src/app/hk-funds/ # Next.js frontend pages

Deployment Model

  • Code-only deployment: Replace .py / .tsx / .ts files. Never overwrite .duckdb
  • Data regenerated on server: All new data (DD scores, NAV, ratings) computed by running pipeline scripts post-deploy
  • Incremental upserts: All pipelines use upsert (INSERT OR REPLACE), safe to re-run
  • Scheduled refresh: Weekly SFC fund/manager refresh + daily enforcement cross-check via hk_funds/scheduler.py

Quick Start (after code deploy)

systemctl restart hk-funds-api # Seed templates + compute ratings python3 -c " from hk_funds.standard_templates import seed_standard_templates from hk_funds.storage import init_db conn = init_db(); seed_standard_templates(conn); conn.close() " python3 -m hk_funds.pipeline_manager_dd curl -X POST http://127.0.0.1:8004/api/v1/templates/6/compute \ -H 'Content-Type: application/json' \ -d '{"user_id":"broker_18i","target_type":"manager"}' # Rebuild frontend cd frontend && npm run build && systemctl restart zt-frontend
